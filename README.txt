PROJECT OVERVIEW
----------------
This is a Laravel 12 application with Inertia.js and React that allows users to:
- Explore historical artifacts from the Met Museum API
- Browse artworks by art movements/eras (Renaissance, Baroque, Impressionism, etc.)
- Save favorite artworks to a personal collection
- View and manage favorites with notes

TECHNOLOGY STACK
----------------
- Backend: Laravel 12 (PHP 8.2+)
- Frontend: React 18 + Inertia.js
- Styling: Tailwind CSS
- Database: SQLite (default) or MySQL/PostgreSQL
- 3D Background: Spline (optional)

PREREQUISITES
-------------
Before you begin, ensure you have the following installed:
1. PHP 8.2 or higher
2. Composer (PHP package manager)
3. Node.js 18+ and npm
4. Git
5. (Optional) Spline account for 3D backgrounds

INSTALLATION STEPS
------------------

STEP 1: Clone the Repository
-----------------------------
git clone https://github.com/zeeke69/laravel-proj-110.git
cd laravel-proj-110

STEP 2: Install PHP Dependencies
--------------------------------
composer install

STEP 3: Install Node Dependencies
----------------------------------
npm install

STEP 4: Environment Configuration
----------------------------------
1. Copy the environment file:
   cp .env.example .env

2. Generate application key:
   php artisan key:generate

3. Configure your .env file with the following settings:

   APP_NAME="Art Movements Expedition"
   APP_ENV=local
   APP_KEY= (generated by key:generate)
   APP_DEBUG=true
   APP_URL=http://localhost:8000

   # Database Configuration (SQLite - default)
   DB_CONNECTION=sqlite
   DB_DATABASE=database/database.sqlite

   # OR use MySQL/PostgreSQL:
   # DB_CONNECTION=mysql
   # DB_HOST=127.0.0.1
   # DB_PORT=3306
   # DB_DATABASE=your_database
   # DB_USERNAME=your_username
   # DB_PASSWORD=your_password

   # Optional: Spline 3D Background URLs
   # VITE_SPLINE_SCENE_URL=your_spline_scene_url
   # VITE_SPLINE_GUEST_SCENE_URL=your_spline_guest_scene_url

STEP 5: Create Database (if using SQLite)
------------------------------------------
touch database/database.sqlite

STEP 6: Run Database Migrations
--------------------------------
php artisan migrate

STEP 7: Build Frontend Assets
------------------------------
For development:
npm run dev

For production:
npm run build

STEP 8: Start the Development Server
------------------------------------
php artisan serve

The application will be available at: http://localhost:8000

RUNNING THE APPLICATION
-----------------------

Development Mode:
-----------------
Option 1: Separate terminals
  Terminal 1: php artisan serve
  Terminal 2: npm run dev

Option 2: Using composer script (recommended)
  composer run dev
  (This runs both Laravel server and Vite dev server concurrently)

Production Mode:
----------------
1. Build assets: npm run build
2. Start server: php artisan serve
3. Optimize: php artisan config:cache
              php artisan route:cache
              php artisan view:cache

FEATURES
--------

1. Dashboard
   - Browse random artifacts from Met Museum
   - Add artifacts to favorites
   - View favorites in draggable panel
   - Edit notes on favorites

2. Art Movements Expedition
   - Filter artworks by era/movement:
     * Renaissance (1400-1600)
     * Baroque (1600-1750)
     * Impressionism (1860-1886)
     * Modern Art (1900-1950)
     * Contemporary (1950-Present)
     * Medieval (500-1400)
     * Ancient Art (Before 500 CE)
   - Add any artwork to favorites
   - Beautiful transparent UI with 3D background

3. Favorites Management
   - View all saved favorites
   - Add personal notes
   - Edit or delete favorites

API INTEGRATION
---------------
This application uses the Met Museum Collection API:
- Base URL: https://collectionapi.metmuseum.org/public/collection/v1/
- No API key required
- Rate limits apply (be mindful of requests)

DATABASE STRUCTURE
------------------
The application uses the following main tables:

- users: User authentication and profile
- favorites: User's saved artworks
  - user_id
  - artifact_id
  - item_name
  - image_url
  - culture
  - object_date
  - notes
  - source

TROUBLESHOOTING
---------------

Issue: "Class not found" errors
Solution: Run `composer dump-autoload`

Issue: Assets not loading
Solution: 
  1. Run `npm run build` or `npm run dev`
  2. Clear cache: `php artisan cache:clear`

Issue: Database errors
Solution:
  1. Ensure database file exists: `touch database/database.sqlite`
  2. Run migrations: `php artisan migrate:fresh`

Issue: Permission errors (Linux/Mac)
Solution:
  chmod -R 775 storage bootstrap/cache
  chown -R www-data:www-data storage bootstrap/cache

Issue: Vite HMR not working
Solution:
  1. Check if port 5173 is available
  2. Ensure VITE_APP_URL in .env matches your local URL

Issue: Met Museum API errors
Solution:
  1. Check internet connection
  2. API may be temporarily unavailable
  3. Check API response in browser network tab

DEPLOYMENT CONSIDERATIONS
-------------------------

For Production Deployment:

1. Set APP_ENV=production in .env
2. Set APP_DEBUG=false
3. Run: php artisan config:cache
4. Run: php artisan route:cache
5. Run: php artisan view:cache
6. Build assets: npm run build
7. Set proper file permissions
8. Use a production web server (Nginx/Apache)
9. Set up SSL certificate
10. Configure proper database (not SQLite for production)

Recommended Hosting Platforms:
- Railway (https://railway.app) - Easy Laravel deployment
- Render (https://render.com) - Good free tier
- Fly.io (https://fly.io) - Great for Laravel
- DigitalOcean App Platform
- Laravel Forge + VPS

Note: Vercel is NOT recommended for full Laravel applications as it's
designed for serverless/static sites. Use platforms that support PHP.
